3 Cool Html 5 Features

New Elements

The new HTML5 elements allow for clear, cleaner code.
In HTML 4 the div was the main element used to divide different sections of the document, the div was usually coupled with a CSS class and that was the way we had to separate logically different sections. This meant that when you looked at your code all you saw where a lot of divs, and it wasn't clear at all what were the different thematic sections of a document.
Now with HTML 5 we have new elements like <header> <footer> <aside> <section> <article> and <nav>.
Just by using this new elements I feel my code is much cleaner and easy to understand for me and for any other developer that might have to work with my code.
Other benefits are accessibility, this makes it easier on screen readers to interpret the web page for disabled people.

Inline editing with contenteditable

Contenteditable is great, I used to add external libraries to my projects to be able to do inline editing. Now with the HTML5 contenteditable attribute is super simple, all you need to do is an attribute to any element and done you have inline editing. Just click on this text and you'll see what I mean.

Audio & Video

HTML 5 introduced native audio and video files support, now all you need to do reproduce a video or play an audio file is to use the new <video> <audio> elements.
